Open Source

Some parts of Continuum are already open. Others will be.

Version 1.0 of Continuum is open source and available here:

https://github.com/andrewgstanton/continuum

The actual application lives under the dashboard/ directory in that repository.

If you want to run Continuum from source, that is the place to start.

Browser Signing Extension

A browser signing extension is also included in this repository under:

browser_extensions/

The extension enables browser-based NIP-07 style signing against a locally running Continuum instance.

Setup instructions are included in the extension README.

Note that the latest Continuum release (v1.6.8.0) is currently distributed separately as a desktop binary or Docker setup and is not open source at this time.

What is not open yet

The version of Continuum I use day to day (continuum-pro) is not public.

That code lives in a separate private repository.

I do plan to open source it.

Once I reach $10,000 in total revenue across Continuum-related work (software, books, patron support, consulting), I will move that code into the main Continuum repository and make it public.

Until then, I am continuing to build and refine it.

Why I am doing it this way

I want Continuum to be something people can actually run and understand.

But I also want the system to be stable enough that when it is opened, it is useful — not confusing or half-finished.

So this is a staged approach:

  • Version 1.0 is open
  • Active development continues privately
  • Full source will be opened once the project is sustainable

That keeps the direction clear without rushing the process.

If you are trying to understand where this is going, the direction is already visible in the code that is public today.
